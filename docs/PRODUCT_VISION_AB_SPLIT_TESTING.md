**Version:** 1.0

## Executive Summary

Verdict will offer two distinct but complementary testing capabilities:

| Feature | A/B Testing | Split Testing |
|---------|-------------|---------------|
| **Purpose** | Quick preference comparison | Demographic-segmented insights |
| **Question** | "Which is better?" | "Which performs better with WHO?" |
| **Output** | Winner + feedback | Segment-by-segment breakdown |
| **Use Case** | Fast validation | Audience-fit discovery |
| **Complexity** | Simple | Advanced |

**Strategic Position:** Combine the speed of PickFu, the qualitative depth of UserTesting, and the demographic precision of Pollfish â€” at accessible pricing with a mobile-native experience.

## Part 2: Feature Definitions

### A/B Testing (Simple Comparison)

**User Story:**
*"As a creator, I want to quickly compare two options and get feedback so I can make a confident decision."*

**Core Flow:**
```
Upload 2 options â†’ Add context â†’ Set verdict count â†’ Pay credits â†’ Get results
```

**Key Characteristics:**
- Fast (results in < 1 hour)
- Simple (2 options only)
- General audience (no targeting)
- Affordable (1-3 credits)
- Clear winner + reasoning

**Output:**
- Winner declaration (A, B, or Tie)
- Vote distribution
- Aggregated feedback themes
- Individual judge reasoning
- Confidence score

---

### Split Testing (Demographic Segmentation)

**User Story:**
*"As a creator, I want to understand how different audience segments respond to my options so I can tailor my approach."*

**Core Flow:**
```
Upload options â†’ Define segments â†’ Set sample size per segment â†’ Pay credits â†’ Get segmented results
```

**Key Characteristics:**
- Deeper (multi-segment analysis)
- Targeted (demographic/psychographic filters)
- Comparative (see differences across groups)
- Premium (5-20 credits)
- Actionable insights

**Output:**
- Per-segment winner
- Cross-segment comparison matrix
- Demographic breakdown visualization
- Divergence alerts ("Photo A wins with women, Photo B wins with men")
- Recommended audience fit

## Part 6: Mobile-First Design Principles

### 1. Thumb-Zone Optimization
- All primary actions in bottom 1/3 of screen
- Swipe gestures for quick decisions
- One-handed operation for all core flows

### 2. Progressive Disclosure
- Show only what's needed at each step
- Advanced options hidden behind "More options"
- Complexity reveals as user needs it

### 3. Instant Gratification
- Live results feed (not waiting for completion)
- Celebration moments on key actions
- Real-time progress indicators

### 4. Offline Resilience
- Draft auto-save
- Queue caching for judges
- Graceful degradation

### 5. Visual Hierarchy
- Images dominate (they're the content)
- Text is supporting, not primary
- Color codes meaning (green = win, etc.)

## Part 9: Technical Architecture Considerations

### Data Model Extensions

```sql
-- Segments for split testing
CREATE TABLE test_segments (
  id UUID PRIMARY KEY,
  test_id UUID REFERENCES tests(id),
  name TEXT,
  demographic_filters JSONB,
  psychographic_filters JSONB,
  target_count INTEGER,
  completed_count INTEGER DEFAULT 0
);

-- Judge demographics (for targeting)
CREATE TABLE judge_demographics (
  judge_id UUID PRIMARY KEY REFERENCES profiles(id),
  age_range TEXT,
  gender TEXT,
  location_country TEXT,
  location_region TEXT,
  interests TEXT[],
  verified_at TIMESTAMP,
  verification_method TEXT
);

-- Segment-level results
CREATE TABLE segment_results (
  segment_id UUID REFERENCES test_segments(id),
  option_id TEXT,
  vote_count INTEGER,
  avg_rating NUMERIC,
  themes JSONB,
  PRIMARY KEY (segment_id, option_id)
);
```

### API Endpoints

```
POST   /api/tests                    # Create A/B or Split test
GET    /api/tests/:id                # Get test with results
POST   /api/tests/:id/segments       # Add segment to split test
GET    /api/judge/queue?demographics # Get targeted queue
POST   /api/verdicts                 # Submit verdict
GET    /api/tests/:id/results        # Get full results
GET    /api/tests/:id/results/segment/:segmentId  # Segment results
```
